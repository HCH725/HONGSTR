# HONGSTR Phase 0: 策略池治理與回測涵蓋率

這是一份寫給非工程營運與策略夥伴的「策略池與版本涵蓋率」指引。  
這裡面的所有機制皆為**唯讀 (Read-only)**，不會干涉實際引擎下單或撮合。

## 1. 什麼是 Strategy Pool？

策略池 (`strategy_pool`) 是一個將「實驗中的策略」過濾、晉升到「實盤備選名單」的管理機制。

- **Candidates (候選名單)**：還在觀察期的策略與參數組合。
- **Promoted (晉升名單)**：連續達到績效指標（如夏普率 > 1.0, 報酬率 > 2%），且表現穩定的策略。
- **Demoted (降級名單)**：曾經優良，但近期表現跌破防線（如連續三次夏普率 < -0.05）的退役策略。

## 2. 什麼是 Coverage Table？

每次執行回測後，系統都會收集對應的成績，並更新到「涵蓋率追蹤表 (`coverage_table.jsonl`)」中。

- **狀態 `DONE`**：回測成功且數據健康。
- **狀態 `NEEDS_REBASE`**：底層的核心語義有過重大升級（例如滑價計算方式變更），導致舊有的回測分數不具參考價值，系統會警告你需要「重新回測」才能算數。

## 3. 升級與降級的防刷機制 (Anti-Churn)

為了避免因為行情隨機波動導致策略「今天上架、明天又下架」，我們設計了滯後門檻 (Hysteresis)：

- 上架門檻高 (`min_sharpe: 1.0`)
- 下架門檻低 (`min_sharpe: -0.05`)
這確保了策略池是相對穩定的池子，而不是每天大風吹。

## 4. 每日怎麼檢查？

營運人員不需要人工維護這個表單！
每次跑完新的 Backtest 後，工程端只要執行：

```bash
python scripts/coverage_update.py
```

當底層引擎有更新時，只要執行：

```bash
python scripts/semantics_check.py
```

全部資料狀態就會自動流轉，並能在 Dashboard 儀表板上直接唯讀查看！
