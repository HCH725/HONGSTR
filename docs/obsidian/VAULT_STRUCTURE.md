# HONGSTR Obsidian Vault Structure

## Purpose

The local Obsidian vault (`_local/obsidian_vault/HONGSTR/`) is the **operator knowledge base** for:

- Incident retrospectives and postmortems
- Ops runbook entries (SOP procedures)
- Decision logs (ADR-lite, change rationale)
- Research experiment records (backtest reproducibility, OOS gates)
- Weekly PM governance reviews

The vault is **never committed to git** (it lives under `_local/`, which is gitignored).  
Templates live in [`docs/obsidian/templates/`](./templates/) and **are** version-controlled.

## Mirror-facing Publish Subset

The iCloud mirror is a curated publish subset, not a full copy of the local vault.

Active mirror contract:

- `Daily/**`
- `Dashboards/**`
- `KB/_meta/**`
- `KB/PR/**`
- `KB/Runbooks/**`
- `KB/Incidents/**`
- `KB/Research-Summaries/**`

Legacy / non-current:

- `KB/SSOT/**` is frozen legacy content and is not part of the active mirror refresh contract.
- Existing `KB/SSOT/**` content on the iCloud target may remain until manually removed because mirror delete mode is disabled.

Optional future surface:

- `State-Refs/**` or `Snapshots/**` may exist only as pointer/readable-summary notes to canonical `data/state/*.json`.
- They must never become a second SSOT or a dependency for `/status`, `/daily`, or `/dashboard`.

This means local folders such as `Incidents/`, `Runbooks/`, or `Research/` can still exist for operator use without automatically becoming part of the mirror contract.

## Naming Conventions

| Folder      | Pattern                           | Notes                          |
|-------------|-----------------------------------|--------------------------------|
| `Daily/`    | `YYYY-MM-DD.md`                   | Auto-generated                 |
| `Incidents/`| `YYYY-MM-DD_<slug-lowercase>.md`  | slug = brief label, e.g. `btc-freshness-gap` |
| `Decisions/`| `YYYY-MM-DD_<topic-lowercase>.md` | topic = decision area          |
| `Research/` | `YYYY-MM-DD_<experiment_id>.md`   | experiment_id matches report dir |
| `Runbooks/` | `<topic-lowercase>.md`            | Stable name; update in-place   |
| `Reviews/`  | `YYYY-WNN_pm_review.md`           | ISO week number                |

## Required Front-matter Fields

Every manually created note **must** include these YAML fields at the top:

```yaml
---
date: YYYY-MM-DD
ssot_ts_utc: ""          # fill from data/state snapshot timestamp
linked_pr: "#NNN"        # GitHub PR number (or "none" with justification)
linked_issue: "#NNN"     # GitHub Issue number (or "none" with justification)
status: draft            # draft | final | deprecated
---
```

Notes missing `linked_pr` or `linked_issue` will surface in the **link audit report**:

```bash
python scripts/obsidian_link_audit.py
# в†’ reports/obsidian_audit/link_audit_YYYYMMDD.md
```

## Auto-Generated vs Manual Notes

| Generator          | Source                     | Output folder   |
|--------------------|----------------------------|-----------------|
| `obsidian_sync.py` | `data/state/*.json`        | `Daily/`, `Strategies/`, `Incidents/` |
| Manual (template)  | Operator writes it         | `Incidents/`, `Decisions/`, `Research/`, `Runbooks/`, `Reviews/` |

For incident notes: auto-generated stubs are created when the system detects WARN/FAIL. You should **copy the postmortem template** and fill it in to replace the stub.
