# Telegram Operator Manual (HONGSTR)

> REFERENCE ONLY
>
> 每日主入口請直接使用 Telegram `/daily`。此文件只作補充排障。

## Daily Report Entry Point

- Command: `/daily`
- Canonical source: `data/state/daily_report_latest.json`
- Output policy:
  - default deterministic fallback template (fixed section order)
  - optional LLM polishing when reasoning client is available
  - on LLM failure/timeout: graceful degrade to fallback with `DAILY_REPORT_STATUS: WARN`

## What `/daily` covers

1. `SystemHealth` (SSOT system health summary)
2. `DataFreshness` (realtime/backtest counts + offenders)
3. `Backtest` (OOS/IS, Sharpe, MDD, Trades, gate)
4. `StrategyPool+Leaderboard` (direction + SHORT coverage + metrics_status)
5. `Governance(Overfit)` (policy version + today gate summary)
6. `Guardrails` (core diff / no-exec / no-data)

每段固定三行：

- `狀態`
- `白話`
- `下一步`

## Acronym policy

`/daily` must explain each acronym at least once in-message:

- `SSOT`, `DD`, `MDD`, `Sharpe`, `Trades`, `OOS`, `IS`, `WF`, `L1`, `L2`, `L3`, `TP`, `SL`, `DCA`

Reference source: `docs/ops/acronym_glossary_zh.md`

## Schedule Ownership (placeholder)

- launchd label placeholder: `com.hongstr.refresh_state`

## Troubleshooting (`/daily` shows missing/unreadable SSOT)

1. Run `bash scripts/refresh_state.sh`
2. Confirm `data/state/daily_report_latest.json` exists and is valid JSON
3. Confirm `data/state/system_health_latest.json` exists
4. Retry `/daily`

If still abnormal, check:

- `logs/launchd_daily_etl.out.log`
- `logs/launchd_dashboard.out.log`
- `data/state/_tg_cp/runtime.log`
