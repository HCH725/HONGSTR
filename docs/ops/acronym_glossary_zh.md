# HONGSTR 術語名詞表 (Glossary)

本手冊定義了 HONGSTR 系統中常見的專用術語，供合作夥伴參考。

| 縮寫 (Acronym) | 中文定義 | 為什麼重要 (Why it matters) | 出現 WARN/FAIL 怎麼辦 |
| :--- | :--- | :--- | :--- |
| **SSOT** | 單一真理來源 (Single Source of Truth) | 確保所有系統組件看到的數據都是一致且最新的。 | 檢查 `refresh_state.sh` 是否運行成功。 |
| **System Health** | 系統健康度 | 衡量基礎設施（數據、剎車、連通性）的整體狀態。 | 若為 FAIL，應暫停所有依賴自動化數據的決策。 |
| **Freshness** | 數據新鮮度 | 指實時數據與當前時間的延遲。 | 若延遲過高，可能導致信號過時，需檢查 ETL 任務。 |
| **Coverage** | 數據覆蓋率 | 檢查回測或訓練所需的歷史數據是否有缺口。 | 若低於 100%，回測結果可能失效，需補齊數據。 |
| **Brake** | 安全剎車 | 一套自動化防護機制，防止異常下單或異常市況。 | 若為 FAIL，代表剎車已觸發，需人工介入檢查。 |
| **Regime Monitor** | 市場環境監測器 | 辨識市場處於趨勢、震盪或極端高壓環境。 | 幫助調整策略參數，但不直接影響存活。 |
| **Regime Signal** | 市場風險訊號 | 系統對開發策略提出的「准許/禁止」開倉訊號。 | **FAIL 代表目前風險高，不進場**；並非系統錯誤。 |
| **Backtest Head** | 回測最前端結果 | 指最近一次模擬回測產出的核心數據指標。 | 用於觀測策略是否因近期市況而產生衰減。 |
| **Gates** | 治理門檻 | 策略從研究到上架必須通過的門檻 (G0-G6)。 | 若 G 級別不夠，不可輕易加大部位。 |
| **Strategy Pool** | 策略池 | 目前「上架中」可隨時供系統調用或監控的策略。 | 主要關注是否有策略因表現不佳被「下架」。 |
| **Leaderboard** | 排行榜 | 潛在候選策略的積分排名。 | 從中選取下一階段推廣至策略池的對象。 |
| **Overfit Gov.** | 過度擬合治理 | 防止策略只對歷史數據有效，對未來無效的機制。 | 若警告亮起，代表策略有「作弊」嫌疑，需打回研究。 |
