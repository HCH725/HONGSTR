# Daily Report Single Entry (/daily)

> PRIMARY ENTRYPOINT
>
> 每日對外只看 Telegram `/daily`。此文件只定義契約與使用方式，其他治理/稽核/runbook 文件皆為 Reference。

## 1) Canonical SSOT

- File: `data/state/daily_report_latest.json`
- Schema version: `daily_report.v1`
- Producer: `scripts/state_snapshots.py`
- Consumers: `_local/telegram_cp/tg_cp_server.py` (`/daily`), dashboard read-only views

## 2) `/daily` 固定輸出模板（口語化 SOP）

報告包含以下 6 個關鍵區段。每一段固定三行：

1. **結論**（狀態: OK/WARN/FAIL）
2. **原因**（簡單、口語的中文說明）
3. **SOP**（建議合夥人或維護者的下一步行動）

### 1. SystemHealth（系統健康）

- **狀態**: `OK`
- **白話**: 系統基礎設施運行良好，所有組件同步正常。
- **下一步**: 無需操作，保持觀察。

### 2. DataFreshness（數據新鮮度）

- **狀態**: `OK`
- **白話**: 市場數據與交易所同步，延遲低於 5 分鐘。
- **下一步**: 若變為 WARN，請檢查 ETL 自動化任務。

### 3. Latest Backtest（最新回測）

- **狀態**: `OK`
- **白話**: 今日自動回測已完成，最新策略表現符合預期。
- **下一步**: 點擊 `[View Artifacts]` 查看詳細損益圖。

### 4. StrategyPool+Leaderboard（策略池與排行榜）

- **狀態**: `OK`
- **白話**: 核心策略池無異動，排行榜前三名表現穩定。
- **下一步**: 考慮將高於門檻的候選策略推廣至小樣。

### 5. Governance / Overfit（治理與過擬合）

- **狀態**: `OK`
- **白話**: 策略實測表现與歷史模擬一致，無過度擬合預警。
- **下一步**: 若變為 FAIL，應暫停該策略併回打研究階段。

### 6. Guardrails（紅線保護）

- **狀態**: `OK`
- **白話**: 程式碼與數據路徑符合安全規範，無非授權變動。
- **下一步**: 確保 PR 通過自動化安全稽核。

## 3) 縮寫對照表 (Glossary for Partners)

旨在幫助合夥人理解報告中的指標與狀態，確保決策依據透明。

| 縮寫（中文解釋） | 它是什麼 | 為什麼重要 | 看到 WARN/FAIL 該做什麼 |
| :--- | :--- | :--- | :--- |
| **SSOT（單一真實來源）** | 系統狀態的唯一權威數據文件。 | 確保各模組看到的數據不打架。 | 執行 `refresh_state.sh` 手動強制刷新。 |
| **MDD（最大回撤）** | 歷史上策略從頂峰跌落的最大百分比。 | 衡量「最倒霉時」會虧多少。 | 若當前虧損 > MDD，應立即停機檢查。 |
| **OOS（樣本外驗證）** | 用策略「沒看過」的數據進行測試。 | 檢驗策略是否具備真實預測能力。 | 若 OOS 與預期差太多，代表策略已失效。 |
| **IS（樣本內測試）** | 用策略「看過」的數據進行參數優化。 | 是研發階段的基礎，但不可作為上線唯一依據。 | 僅供研究員參考，合夥人可忽略。 |
| **WF（步進驗證）** | 模擬真實時間推移的滾動測試。 | 比靜態回測更接近實盤交易。 | 是策略晉升至 Gate 4 的核心標準。 |
| **TP（停利點）** | 預設的獲利離場位置。 | 確保入袋為安，避免獲利回吐。 | 一般由算法自動執行，不需手動調整。 |
| **SL（停損點）** | 預設的止損離場位置。 | 留得青山在，避免單次爆倉風險。 | 這是最後一道防線，嚴禁手動取消。 |
| **DCA（分批成交計畫）** | 定期定額或跌幅倍數加碼的進場法。 | 降低平均持倉成本，平滑波動。 | 需定期檢查 `base_notional` 設置。 |
| **L1/L2/L3（風險/成本分層）** | 描述交易成本與滑價的嚴苛程度。 | 決定策略在真實市場是否還有利潤空間。 | L3（最嚴）若虧損，代表策略實盤不可行。 |
| **Sharpe（夏普比率）** | 風險調整後的報酬指標。 | 衡量每承擔 1 分風險能換回多少利潤。 | 理想值應大於 1.5。若變低，代表 CP 值下降。 |
| **DD（當前回撤）** | 從近期損益頂峰跌下來的幅度。 | 反映當前市場對該策略的是否友善。 | 若 DD 持續擴大，應進入觀望模式。 |
| **Gate（上架門檻）** | 策略晉升的 0 到 6 級評估標準。 | 確保只有高品質、高穩定的策略能上場。 | 未達 Gate 4 的策略嚴禁投入實資金。 |
| **Regime（市場環境）** | 區分牛市、熊市或震盪市。 | 讓合夥人知道現在的天氣適合哪種農作（策略）。 | 關注 `regime_slice` 是否匹配策略標籤。 |
| **Fallback（降級機制）** | 系統故障時自動回退至安全預設值。 | 確保系統在異常時「優雅停下」而非亂跑。 | 若頻繁 Fallback，代表規則需要重修。 |
| **Brake（安全剎車）** | 系統在損失過大或數據異常時自動觸發。 | 防止系統失控造成毀滅性損失。 | FAIL 時必須由維護者排錯後手動重啟。 |

## 4) 缺值規則（硬性）

- 指標缺值不得顯示 `0.00`
- 一律使用：`null + metrics_status=UNKNOWN + metrics_unavailable_reason`
- `/daily` 顯示為：`資料不足/UNKNOWN`

## 5) LLM 潤飾策略

- 預設先生成 deterministic fallback（固定模板）
- 若 reasoning client 可用，僅做文字潤飾
- 若 LLM 失敗或 timeout：`DAILY_REPORT_STATUS: WARN`，並回退 deterministic fallback
