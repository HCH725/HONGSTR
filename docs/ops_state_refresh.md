# HONGSTR 一鍵狀態刷新與儀表板快照指南 (Ops State Refresh)

## TL;DR (太長不看版)

**`refresh_state.sh` 是一支「安全、唯讀、免動腦」的一鍵總結腳本**。
它**不會**擅自幫你下單，也**不會**改變核心撮合邏輯。它純粹負責去「掃描現有的各種散落報告」，把最新的數據整理成 Dashboard 喜歡的「摘要小檔 (`.json`)」，讓你看總表時系統又快又順。

---

## 什麼時候該跑這支腳本？

這支腳本**不上自動排程 (No scheduling)**。當你完成以下兩種情境，想要 Dashboard 更新時，手動執行即可：

1. **剛跑完大型回測或日常訓練**，你想馬上讓網頁版看到最新的覆蓋率和分數。
2. **手動更新了各種 Config 設定**，你想確認策略池名單最新的晉級/降級狀態。

## 執行指令

```bash
bash scripts/refresh_state.sh
```

## 執行完了，會有什麼變化？

它只做了一件事：「讀取 -> 產出三大快照」。產生的東西全部會乖乖收在 `data/state/` 資料夾內，作為網頁顯示用的暫存檔。

### 1. `coverage_latest.json`

原本的歷史追蹤紀錄（`jsonl`）可能是好幾百萬筆一直疊加上去的流水帳。這個檔案會幫你**去蕪存菁**，對於同一個幣種跟時區，只保留「最新鮮的一筆」。

### 2. `coverage_summary.json`

這是一個給首頁看的小字卡，只留了：

- **通過率 (pass_rate)**：所有回測裡有幾成是及格的。
- **整體平均夏普值 (avg_sharpe)**
- **涵蓋數 (count)**

### 3. `strategy_pool_summary.json`

這是 Dashboard 右側「排行榜」的專用資料檔。它已經幫你算好了：

- 目前共有幾位「候選人 (Candidates)」、幾位「晉升者 (Promoted)」。
- 只抓取前 5 ~ 10 名績效最好的選手，減輕前端瀏覽器負載。
